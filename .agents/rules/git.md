# git

Basic rules for procdures when doing work with version control using git.

## New Work **REQUIRED**

When starting new work, create a new branch from the main branch. The branch name should be descriptive of the work being done, and follow the format `feature/description` for new features, `bugfix/description` for bug fixes, and `hotfix/description` for urgent fixes.


## Submitting PRS

When submitting a pull request, ensure that the PR description is clear and concise, outlining the changes made and the reason for the changes. Include any relevant issue numbers or links to related work. Additionally, ensure that the PR is linked to the appropriate project board and that all necessary labels are applied.

Utilize the `gh` CLI tool to create pull requests, as it allows for better integration with GitHub and can help automate some of the processes involved in PR creation and management. This can help ensure that all necessary information is included in the PR and that it is properly linked to the relevant project board and labels.
